Response Info

All successful responses are wrapped in a common json response. This common json response consists of a 'status', 'status_code' and 'resp_data' keys. The 'status key' is a string stating the successful action that just took place. By default this is "Success". 'status_code' is an integer which for successful responses is always 0. The 'resp_data' key holds all the pertinent data for the action that just took place in result of the api call. This will be a json object and the contents are specified per method below in the docs.
